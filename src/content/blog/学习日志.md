### 2024.12.26

关于Linux */opt* 目录与 */usr/local*目录

- FHS将/usr/local定义为"供系统管理员在本地安装软件时使用"。这可能会令人困惑，因为此描述与/opt非常相似。另一方面，有一个重要的区别。
- /usr/local下的层次结构应该模仿/usr下的层次。这意味着我们应该将所有应用程序二进制文件放置在/usr/local/bin中，将所有库放置在/usr.local/lib中，以此类推。因此，我们将把它们与其他应用程序的文件一起放置。
- 因此，在部署到/usr/local时，我们不能为每个应用程序都设置一个目录。相反，我们以更传统的UNIX风格组织它们，将应用程序文件划分为单独的目录。
```
The FHS defines /usr/local as "For use by the system administrator when installing software locally". This may be confusing, as this description is very similar to /opt. On the other hand, there's an important difference.

The hierarchy under /usr/local should mimic the hierarchy under /usr. This means we should place all the application binaries in /usr/local/bin, and all the libraries in /usr/local/lib, and so on. As a result, we'll place them along with the files of the other applications.

Therefore, we can't have a single directory for each application when deploying into /usr/local. Instead, we organize them in the more conventional UNIX style that divides application files to separate directories.
```

### 2024.12.4

C# 抽象类 和 抽象方法
1. **抽象方法：**
当你使用abstract关键字声明一个方法时，这意味着该方法在抽象类中不提供实现。任何继承该抽象类的子类都必须重写（实现）这个抽象方法，除非子类本身也被声明为抽象类。
2. **抽象类：**
当你使用abstract关键字声明一个类时，这意味着该类不能被实例化。它必须被其他类继承。抽象类可以包含抽象方法（没有实现的方法），也可以包含具体方法（有实现的方法）。
继承抽象类的子类可以选择实现所有的抽象方法，从而成为一个具体类，或者选择不实现某些抽象方法并继续保持为抽象类。

C# 隐藏 和 重写

1. **访问修饰符**：隐藏的函数可以具有与父类中同名函数不同的访问修饰符（如public、protected、private等）。
2. **参数列表和返回类型**：隐藏的函数可以与父类中同名函数具有不同的参数列表或返回类型（这实际上会构成方法重载，而不是隐藏，但如果参数列表和名称相同但返回类型不同，这在C#中是不允许的，除非使用new关键字显式隐藏）。
3. **多态性**：重写的函数保持了多态性，即当你通过父类引用但指向子类实例时，调用的是子类中的实现。而隐藏的函数则不会，通过父类引用调用时仍然会调用父类中的实现。
4. **new和override关键字**：在子类中隐藏父类中的函数时，通常会在子类的函数声明前使用new关键字来显式指明这是一个隐藏操作（尽管在C#中，如果子类中的函数与父类中的函数仅在返回类型上不同，则必须使用new）。而重写父类中的虚函数时，则使用override关键字。
5. **函数签名**：重写要求子类中的函数与父类中的虚函数具有完全相同的签名（包括名称、参数列表和返回类型）。隐藏则允许签名不同，但名称相同通常会导致隐藏（除非构成重载）。

#define N
#define LL long long

vector<int> func1(vector<int>res){
    //容器内值不变
    for(auto i:res){
        ++i;
    }
    //容器内值会变
    for(auto& i:res){
        ++i;
    }
    return res;

    for(auto& i:res){
        ++i;
    }

vector<int>& func3(vector<int>&res){
    for(auto& i:res){

    }

    for(auto& i:res){
        ++i;
    }

    for(auto i:res){
        cout<<i<<" ";
    }
    cout<<endl;
}
struct node{
    int a,b;
    node(int _a,int _b):a(_a),b(_b){ }
    void display(){
        cout<<a<<" "<<endl;
    }
};
int main(){
    vector<int>a({1,2,3 });
    vector<node>q;
    display(a);
